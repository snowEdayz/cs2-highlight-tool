# Move Runtime Data To LocalAppData

## Goal

Move application-managed runtime data out of the executable directory so users can run the downloaded executable from Downloads without creating many sibling folders and files. On Windows, the default data root should be `%LOCALAPPDATA%\CS2 Highlight Tool\`. This task does not change Wails/WebView2's own cache/user-data directory.

## What I Already Know

* The app currently resolves `exeDir` from `os.Executable()` in `internal/app/app.go`.
* `exeDir` is passed into `envsetup.New`, stored as `App.exeDir`, and reused as the root for config, components, demos, temp files, updates, projects, and outputs.
* Startup currently creates working directories under `exeDir`: `temp`, `hlae`, `plugin`, `ffmpeg`, `updates`, and `demo/raw`.
* `config.Default(exeDir)` currently sets default managed paths under `exeDir` for HLAE, plugin DLL, FFmpeg, and outputs.
* The user wants to implement the `dataDir = %LOCALAPPDATA%\CS2 Highlight Tool\` direction first and leave the Wails/WebView2 cache directory unchanged for now.

## Assumptions

* This change targets Windows packaging/runtime behavior first. Non-Windows platforms should keep a sensible fallback and continue to pass tests.
* `exeDir` still matters for self-update because the updater replaces the executable at its current path.
* Business/runtime data should be grouped under one app-owned data root.
* No frontend UI changes are required unless a path exposed to the UI changes shape.

## Open Questions

* None.

## Requirements

* Add a data-directory concept distinct from executable directory.
* On Windows, resolve default data root to `%LOCALAPPDATA%\CS2 Highlight Tool\`.
* Move app-managed generated/downloaded data to the data root:
  * `config.json`
  * `hlae/`
  * `plugin/`
  * `ffmpeg/`
  * `temp/`
  * `updates/`
  * `demo/`
  * `projects/`
  * `outputs/`
  * startup log exports default directory
* Keep self-update executable replacement anchored to the executable path.
* Do not change Wails/WebView2 `WebviewUserDataPath` in this task.
* Do not manually edit generated frontend binding files.
* On first run after upgrade, migrate existing app-managed data from the old executable-directory layout into the new data root.
* Migration should be conservative:
  * migrate only known app-managed paths;
  * do not overwrite existing destination files/directories;
  * prefer moving when safe, but fall back to copying if moving fails;
  * leave source data intact if migration cannot be completed safely;
  * never delete unrelated files next to the executable.
* Migration should cover:
  * `config.json`
  * `hlae/`
  * `plugin/`
  * `ffmpeg/`
  * `updates/`
  * `demo/`
  * `projects/`
  * `outputs/`
  * `logs/`

## Acceptance Criteria

* [ ] Running the app from an arbitrary directory no longer creates app-managed component/demo/temp/config folders next to the executable.
* [ ] Startup checks install/check HLAE, plugin, and FFmpeg under the data root.
* [ ] Demo imports and match imports store managed demo files under the data root.
* [ ] Clip settings and projects use the data root for persisted state and default outputs.
* [ ] Self-update still downloads update artifacts to the app data root but replaces the executable at its executable path.
* [ ] Existing old-layout data is migrated into the data root without overwriting newer data.
* [ ] Backend tests cover path resolution and key data-root consumers.

## Definition Of Done

* `go test ./...` passes.
* If frontend bindings/types are regenerated by Wails due to exposed API changes, generated files are produced by the generator only.
* AGENTS docs are updated if stable path-contract wording changes.

## Out Of Scope

* Changing Wails/WebView2 cache/user-data location.
* Building or modifying the NSIS installer.
* Adding a full portable-mode toggle unless explicitly approved as part of this task.
* Moving user-selected CS2 install paths.

## Technical Notes

* Likely impacted backend files:
  * `internal/app/app.go`
  * `internal/envsetup/service.go`
  * `internal/envsetup/service_startup.go`
  * `internal/config/config.go`
  * app methods that currently call `config.LoadOrCreate(filepath.Join(a.exeDir, "config.json"), a.exeDir)`
  * import/produce/project helpers that currently use `a.exeDir`
  * updater calls that currently receive only `exeDir`
* Current repo has pre-existing uncommitted changes, including generated `frontend/wailsjs/**`; this task should not revert or manually edit them.
